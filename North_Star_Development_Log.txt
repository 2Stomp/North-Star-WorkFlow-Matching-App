NORTH STAR — DEVELOPMENT LOG
================================

This file tracks what exists in the app and what was introduced over time.
Keep this file updated as the single source of truth for features and decisions.

--------------------------------------------------
PROJECT OVERVIEW
--------------------------------------------------
App Name: North Star
Core Idea:
A workflow-first collaboration app with optional discovery and matching.
Projects are the center. Discovery is secondary and intentional.

Core Principles:
- Workflow before matching
- Projects before people
- Calm, intentional UI (no dark patterns)
- Free by default, value-aligned monetization
- Backend-ready architecture

--------------------------------------------------
CORE FEATURES (CURRENT STATE)
--------------------------------------------------
• Create Project flow
• Project hubs (members, channels, tasks, settings)
• Roles & permissions: Owner / Admin / Member
• Project-only chat (no DMs)
• Default channels (Main, Coding, Design, Marketing)
• Custom channels (Star projects)
• System messages (joins, leaves, role changes, task updates)
• Project archive (read-only)
• Leave project with ownership transfer guard
• Local persistence (state survives app restarts)
• Backend-ready AppState design

--------------------------------------------------
DISCOVERY & MATCHING
--------------------------------------------------
• Swipe-based discovery (Projects ↔ People)
• Intent-based matching (no auto-matches)
• Direct invites (friends / coworkers)
• Invite notes
• Selected-project invites
• Lock / unlock project from discovery
• Project visibility controls (what shows on swipe)
• Profile visibility controls (what shows on swipe)
• User intent mode (Looking / Building / Browsing)
• Project stage (Idea / In Progress / Polishing / Maintenance)
• Roles needed per project
• “Why you’re seeing this” transparency

--------------------------------------------------
TASK SYSTEM
--------------------------------------------------
• Owners/Admins can create tasks
• Tasks include:
  - Title
  - Description
  - Due date
  - Assignees
• Tasks appear:
  - In Main channel
  - On Home dashboard
  - In Project Tasks screen
• Assignees can:
  - Mark complete
  - Add completion notes
  - Attach files/images
• Owners/Admins approve task completion

MONETIZATION — STAR TIER
--------------------------------------------------
Star (Pro) Philosophy:
Reward commitment, never punish free users.

Star Projects:
• ⭐ Star badge
• Custom channels
• Project branding hooks
• Expanded role expectations
• Project insights-lite
• Priority discovery (clearly labeled)
• Higher invite limits
• Restore archived projects

Star Individuals:
• ⭐ Star badge (profile, swipe, chat)
• Profile boost (transparent)
• Advanced visibility controls
• Priority invite inbox

No ads. No pay-to-win. No pressure.
